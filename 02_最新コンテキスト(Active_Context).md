## 6. Pain Threshold Filter（痛みの許容範囲によるフィルタリング設計）

> [!IMPORTANT]
> **ポスト・企画・商品設計時に必ずこのフィルタを適用すること**
> クレームや問題行動は「顧客のモラル」ではなく「痛みの許容範囲」の問題である。

### 核心原理
**「その金額が"痛い"人ほど、感情的になる」**

| 失敗時の反応 | 原因 |
|-------------|------|
| クレームが来る | その投資額が「生活に響く」層が来ている |
| 変な人が来る | 「一発逆転」に人生を賭けている（失敗許容ゼロ） |
| コミュニティが荒れる | 期待と現実のギャップが「痛み」として直撃 |

### 価格帯と顧客心理の法則

```
【低価格帯】→ 集客◎ / クレーム◎ / 依存◎
  - その金額が「痛い」層が主要顧客
  - 失敗 ＝ 自己否定 → 感情爆発
  - 「お前のせいで損した」という攻撃に発展

【高価格帯】→ 集客△ / クレーム✕ / 自走◎
  - 失敗しても「誤差」の層が主要顧客
  - 期待値が「体験・好奇心」に向かう
  - 結果への執着が薄い → 問題が発生しにくい
```

### コンテンツ・企画作成時の適用ルール

**① 集客段階（ポスト・発信）:**
- 「藁にもすがる」層ではなく「余裕ある挑戦者」を引き寄せる言葉選び
- NG：「これで人生変わる」「最後のチャンス」「誰でも」
- OK：「実験できる人向け」「失敗を楽しめる人向け」「余剰資金で試す」

**② 商品・企画設計:**
- 価格設定 ＝ 顧客フィルタリング装置として機能させる
- 高単価（30-50万）は「失敗耐性のある層」を自動選別
- 「この金額が痛いなら来るな」というスタンスを暗に示す

**③ 参加条件の明示:**
- 「100%の成功保証はない」ことを前提として提示
- 「自己責任で判断できる人」を明確にフィルタ
- これは「逃げ」ではなく「構造設計」

### 例：フィルタリングを効かせた表現の比較

| NG（痛い層を集める） | OK（余裕層を集める） |
|---------------------|---------------------|
| 「人生最後のチャンス」 | 「実験したい人向け」 |
| 「誰でも成功できる」 | 「失敗を糧にできる人向け」 |
| 「この価格は今だけ」 | 「この投資が誤差の人向け」 |
| 「100%結果を保証」 | 「100%動く武器をわたす、使うのは君だ」 |

> [!TIP]
> **武器商人のスタンス**: 武器を売る。使い方は教える。だが戦果は本人次第。
> このスタンスを一貫することで、依存体質・クレーム体質の層を構造的に排除する。
